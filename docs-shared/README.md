# docs-shared 共享文档根目录

本目录只存放需要进入 Git 流程的共享文档，用于沉淀需要评审、回溯、协作的内容。

## 收录原则

- 需要多人查看、评审或持续维护
- 需要和代码版本一起追踪变更
- 需要为后续迭代提供背景、决策记录或交付说明

例如：

- 结构重组后的共享约定
- 后端是否拆分多项目的决策记录
- 需要多人同步的接口或运行说明

## 不建议放在这里的内容

- 仅供个人使用的临时草稿
- 本地调试笔记或一次性记录
- 不适合进入 Git 历史的个人材料

例如：

- 个人排查过程中的临时命令摘录
- 尚未整理完成的思路草稿
- 只对本机环境有效的私有说明

## 与 `Docs/` 的边界

`Docs/` 继续作为本地专用目录使用，不纳入 Git，也不是仓库内共享文档入口。  
如果某份文档需要进入评审、回溯或协作流程，请整理后放入 `docs-shared/`。

## 后续扩展

后续可以根据实际协作需要，在本目录下逐步扩展共享文档结构。  
在新增具体子目录或共享文档之前，请先以本说明作为当前入口。
